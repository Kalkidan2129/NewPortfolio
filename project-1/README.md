
# Walmart Store Sales

## Project Overview

This project involved creating a comprehensive Power BI dashboard to analyze sales data from 45 Walmart stores in the US between 2010 and 2012. It focused on year-over-year sales performance during the post-2008 recession period, incorporating data modeling, DAX measures, and conditional columns to uncover sales trends.

---

## Business Problem

Walmart stores faced sales fluctuations during the recovery from the 2008 recession, impacting revenue and operational planning. Understanding these sales trends was critical to optimize inventory and marketing strategies. The challenge was to analyze multi-year sales data effectively to identify key factors influencing store performance.

---

## Objective

- Analyze year-over-year sales trends across multiple Walmart stores
- Create a dynamic Power BI dashboard for insightful visualizations
- Incorporate economic and seasonal factors such as holidays and unemployment rates

---

## Tools & Technologies

- Power BI Desktop
- DAX (Data Analysis Expressions)
- Power Query Editor
- Excel (for initial data handling)
- CSV data files

---

## Project Workflow

- Import Walmart sales CSV data into Power BI Desktop
- Create calendar and refresh date tables for time intelligence
- Establish relationships between sales and calendar tables
- Develop conditional columns for holidays and unemployment rates
- Build DAX measures to calculate total, average, and year-over-year sales

---

## Key Insights

- Sales were impacted by the economic recovery period following the 2008 recession
- Holiday weeks showed distinct sales patterns compared to non-holiday periods
- Unemployment rate influenced store sales performance across different regions
- Year-over-year sales trends varied significantly among the 45 Walmart stores
